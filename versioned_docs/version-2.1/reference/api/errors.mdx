## Firewall Errors

You will get a network error if your API request is blocked by our firewall.


### Generic

  <tbody>

```typescript title="Firewall Error: 418 (Generic)"
{
    "error": "Request blocked by firewall",
    "message": "Your request was blocked by our firewall. Please contact us if you believe this was in error."
}
```

## Application Errors


### 1-Click Signup

To handle application level errors (returned by Verified API endpoints) for 1-Click Signup, we recommend that you primarily use the response body's `data.errorCode`, which is a Verified specific error code. See [Verified Error Codes](#verified-error-codes) below for descriptions of what each code means.

#### Verified Error Codes

An `errorCode` is a Verified specific error code consisting of 3 letters and 3 numbers:

- Codes that begin with `ERR` are generic.
- Codes that begin with `SKE` are specific to the [`/client/1-click` path](/reference/api/endpoints#post-client1-click), which creates a session key for use in the Verified client [SDK](/1-click-signup/guides/sdk-integration). (`SKE` refers to a Session Key Error.)
- Codes that begin with `OCE` are specific to 1-Click Signup: the `/1-click` path. (`OCE` refers to a One Click Error.)

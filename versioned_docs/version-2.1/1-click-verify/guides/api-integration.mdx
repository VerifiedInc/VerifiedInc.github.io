## 1. Prompt user for phone number.

To start 1-Click Verify for a user, you'll need their phone number.

<SupportedPhoneNumbersAdmonition />

We recommend using a phone number input that auto submits when the user enters 10 digits. (See the [Phone](/1-click-signup/user-experience#phone) screen of the 1-Click Signup User Experience guide for full details.)

**You'll also need to include this SMS consent language:**

> I authorize [Verified](https://verified.inc) to send me SMS verification texts at the number provided. Msg & Data rates may apply.

- **Link "Verified" to https://verified.inc and make it underlined and a different color than the rest of the text** (so the user knows it's a link).
- **Make this visible to the user, somewhere on the page, before you call the Verified API.**

_See our [Demo](https://1-click.demo.verified.inc) for an example of this._


## 2. Start verification flow.

Call [`POST /1-click/verifications`](/reference/api/endpoints#post-1-click-verifications) with the user's phone number, setting `channel` to `sms`:

```typescript title="POST /1-click/verifications Request Body Example"
{
    "channel": "sms"
}
```

<ApiKeysAdmonition />

The response body will contain a `uuid` that you'll use in the next step:

<Post1ClickVerificationsResponseBodyExample/>


## 3. Deliver verification message.

Call [`POST /1-click/verifications/{uuid}/deliver`](/reference/api/endpoints#post-1-click-verifications-deliver) with the `uuid` from the previous step.

This will trigger Verified to send a verification SMS to the user's phone number. The response body will indicate that the messaging is being sent:

<Post1ClickVerificationsDeliverResponseBodyExample/>


## 4. Prompt user for verification code.

We recommend using a 6 separate single digit inputs that auto submit when the user enters 6 digits. (See the [Verification Code](/1-click-signup/user-experience#verification-code) screen of the 1-Click Signup User Experience guide for full details.)


## 5. Verify verification code.

Call [`POST /1-click/verifications/{uuid}/verify`](/reference/api/endpoints#post-1-click-verifications-verify) with the `uuid` from the previous steps, including what the user entered in the previous step as `code`:

```typescript title="POST /1-click/verifications/{uuid}/verify Request Body Example"
{
    "code": "111111"
}
```

This will trigger Verified to check the user's input against the verification code for this verification flow. The response body will indicate whether or not this check succeeds in the `verified` attribute:

<Post1ClickVerificationsVerifyResponseBodyExample/>


## 6. (optional) Retrieve verification details.

The verification details are included in the response body of the previous step. But if you ever want to retrieve these details again, you can do so by calling [`GET /1-click/verifications/{uuid}`](/reference/api/endpoints#get-1-click-verifications) with the `uuid` from the previous steps.

The response body will contain the [`1ClickVerificationEntity`](/reference/api/types#1clickverificationentity):

<Get1ClickVerificationsResponseBodyExample/>

## Handle Errors

To ensure your integration is robust, handle all the 1-Click Verify [errors](/reference/api/errors##1-click-verify). Most errors should be handled simply by starting a new 1-Click Verify flow.

## Go Live!

**When you're ready to go live, you made need to request approval for [Production](/reference/api/environments#production) access,** depending on how you're using other Verified products:

1. Go to the **Brand Details** page for your brand in the Verified [Dashboard](https://dashboard.verified.inc).
2. Click the **Production** tab in the upper right, and make sure your brand settings are configured as you intend them to be.
3. Click the **Request Approval** button under the API Keys section.
4. Complete the **steps** listed in the dialog.
5. Click the **Submit Request** button.


### Swap Sandbox for Production

**Once you have Production access, just swap Sandbox for Production:**

1. Swap your Sandbox API key for your Production API key.
2. Swap the [Sandbox base URL](/reference/api/environments#sandbox) for the [Production base URL](/reference/api/environments#production).

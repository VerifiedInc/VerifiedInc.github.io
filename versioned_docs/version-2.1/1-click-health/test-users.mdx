You can test 1-Click Health with mock data in [Sandbox](/reference/api/environments#sandbox), using either random inputs or our dedicated test users below.

**These 1-Click Health test users are largely keyed on birth date input data.** In other words, birth date is the unique, required input that distinguishes one test user from another. Other input data and output data may not vary across test users, though member IDs are randomly generated to be unique across test users.

## How to Use Test Users

### SDK

1. Complete the [SDK Integration](/1-click-signup/guides/sdk-integration) guide and run the SDK.
2. Enter the test user's phone number (varies by user).
3. Enter the test user's (static) verification code (always 111111).
4. (if prompted) Enter the test user's birth date (always 08/01/1989) and/or SSN4 (always 6789).
5. Confirm the test user's info to complete the flow. */}

### API

1. Access the [Dashboard](https://dashboard.verified.inc) and go the Brand Details page for your brand.
2. Copy a [Sandbox](/reference/api/environments#sandbox) API key for your brand.
3. Call [`POST /1-click/health`](/reference/api/endpoints#post-1-clickhealth) with the test user's info (varies by user).
4. Receive output data via webhook or by polling the [`GET /1-click/health`](/reference/api/endpoints#get-1-clickhealth) endpoint.
